# Contributing to Receipt Standard

Thank you for your interest in improving the Receipt Standard.

## How to Contribute

We welcome contributions in the following areas:

**Specification Improvements**
- Clarifications to the schema
- Additional field definitions
- Use case documentation

**SDK Enhancements**
- Bug fixes in TypeScript or Python libraries
- Performance improvements
- Additional validation rules

**Framework Examples**
- Integration examples for new frameworks
- Improved existing examples
- Documentation improvements

**Language Support**
- New SDK implementations (Go, Rust, Java, etc.)
- Translations of documentation

## Guidelines

**Code Quality**
- Follow existing code style
- Add tests for new functionality
- Update documentation

**Specification Changes**
- Open an issue first to discuss
- Ensure backward compatibility
- Provide migration path if breaking

**Examples**
- Keep examples simple and focused
- Include comments explaining key concepts
- Test examples before submitting

## Submission Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests if applicable
5. Update documentation
6. Submit a pull request

## Pull Request Template

```
## Description
Brief description of changes

## Type of Change
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation update
- [ ] Specification clarification

## Testing
How were these changes tested?

## Checklist
- [ ] Code follows project style
- [ ] Tests added/updated
- [ ] Documentation updated
- [ ] Backward compatible (or migration path provided)
```

## Code of Conduct

Be respectful and constructive. We're building a standard for the community.

## Questions?

Open an issue or email spec@jakvan.io
